# huitun 元数据模式拆解示例

> 输入：`huitun_search(niche='字节设计面试,腾讯设计岗', top=3, date_range='7d')` 的采集结果
> 历史版本（v0.3.1 推断模式）案例，v0.3.2 已撤销该路径，仅供回顾

## 背景

从 huitun 拉取 3 条笔记后，CES 最高的笔记（「暑期招聘来了，无偿，但需要67月到岗」，CES=5235）已 404 无法访问。
第二条笔记（「给大家普及一下腾讯AI产品经理社招要达到的」，CES=2692）同样 404。
第三条（「0代码经验也能做的可交互式html简历」，CES=520）可访问但 CES 太低。

按 v0.3.1 降级规则：选中 CES 第二高的实用干货型笔记做元数据模式分析。

## 标题驱动分析

标题：「给大家普及一下腾讯AI产品经理社招要达到的」

- 钩子类型推断：身份背书型（"给大家普及"暗示知情人）+ 痛点共鸣型（锁定腾讯AI产品经理求职者）
- 目标人群明确：想进腾讯做AI产品的求职者
- 缺攻击性：开场是平铺式"普及"，不是攻击式假设

## 路飞适用性评分

- 选题契合度：⭐⭐⭐⭐⭐ 大厂求职赛道完全重叠
- 格式迁移性：⭐⭐⭐⭐⭐ 路飞只有视频，没做图文工具型——这是最大格式缺口
- CES启发：转发贡献26.4% → "大厂岗位标准清单"类内容天然有转发力
- 博主壁垒：4粉做到911收藏 → 纯选题红利，路飞背书更强会放大效果
- 综合评分：8.5/10
- 核心建议：路飞应补图文工具型「字节/腾讯设计岗面试要达到的标准清单」
