# 小红书爆款逐字稿生成

## 何时使用本 SKILL

当用户给你：
- 一个选题方向（如 "字节作品集面试模拟"）
- 一份或多份 `xhs-viral-analysis` 拆解报告（作为参考样本）
- 博主画像（如：路飞 = 前滴滴/VIPshop 设计总监，15 年面试官经验，沉浸式模拟面试系列）

并要求：
- "帮路飞写下一条逐字稿"
- "按这个选题方向出一版稿子"
- "复刻这个爆款的逻辑给路飞用"

**不要在以下情况使用**：
- 用户给的是空的选题（"帮我写点设计相关的"）→ 先用 `xhs-topic-selection` SKILL 定方向
- 用户没给拆解报告或参考样本 → 让 agent 先调 `xhs-viral-analysis` 拆 1-3 条参考样本，否则生成是闭门造车
- 用户要的是营销文案 / 公众号长文 / 短视频脚本（带分镜的） → 本 SKILL 只生成"博主对镜说话型"逐字稿
- 用户没给博主画像 → 强制询问，没人设的稿子是泛稿，必流水

## 输入契约

**必填**：
- `topic`: 选题方向（如 "字节 UX 设计岗作品集面试模拟"）
- `note_type`: 笔记类型，二选一：`video`（逐字稿）/ `graphic`（图文笔记）
- `creator_persona`: 博主画像，至少包含：
  - 真实背景（前 X 公司什么职位）
  - 资历数字（N 年 X 经验）
  - 已有爆款配方（如 "沉浸式模拟面试 vol.X 系列"）
  - 服务的受众痛点（如 "设计师求职焦虑"）

**强烈建议**：
- `reference_breakdowns`: 1-3 份 `xhs-viral-analysis` 拆解报告（用于复刻成功配方）
- `target_threshold`: 目标爆款档（按 xhs-viral-analysis §第五维 矩阵）
- `niche`: 赛道（默认 设计师/职场求职）

**可选**：
- `series_number`: 系列编号（如 "vol.14"，触发系列复利效应）
- `target_company`: 目标公司（字节/腾讯/美团/...，越具体钩子越强）
- `target_role`: 目标岗位（UX/UI/IXD/...，越具体越好）
- `prior_works`: 博主已发布的同系列稿子（避免重复 + 保持文风一致）

## 生成流程（Agent-first，不是 pipeline）

⚠️ 这不是流程图，是 agent 看到 goal 后应该自主做的决策序列。如果 context 缺数据，agent 应该主动问，不要硬编。

### Step 1: 准备阶段（必读拆解报告，禁止跳过）

**Agent 必须先在 context 内"消化"**：
1. 读完所有 `reference_breakdowns`，提取爆款共同公式（钩子类型 / 框架类型 A/B/C / 爆点模板 / 情绪曲线特征）
2. 对比博主的成功配方 vs 失败配方（看博主有没有同 niche 流水案例做反例）
3. 识别赛道特异性约束（按 xhs-viral-analysis 的赛道特异性表）
4. 确定目标框架类型（A/B/C 三选一，**一旦选定后不准混合**）

**判定规则**：
- 如果博主有 ≥ 1 条爆款用类型 B 情境再现型 → 强制类型 B（不要试新格式）
- 如果选题是"教程/攻略/清单" → 类型 A 或 C
- 如果选题是"模拟/对话/第一视角" → 类型 B
- 路飞场景默认类型 B（情境再现型）

### Step 2: 生成阶段（按笔记类型分支）

#### 视频笔记（`note_type=video`）

**必须按以下骨架生成**，每段标注秒数预期（按 4-6 字/秒口播速度估算）：

```
[标题候选]
- 生成 3-5 个标题，每个必须满足"标题三件套"：{数字/系列编号} + {具体公司} + {具体类型}
- 例：vol.14 字节 UX 作品集深挖面试模拟 / 154 秒沉浸式腾讯交互设计面试

[逐字稿]

【段 1: 钩子开场 (3-15 秒)】
- 严格用 xhs-viral-analysis 7 种钩子类型之一
- 类型 B 路飞场景：**强制用攻击式开场**（侵略性假设问题）
- ❌ 严禁："大家好我是..." / "今天给大家分享..." / "为什么有的人 X 有的人 Y..."

【段 2: 情境/价值展开 (主体 70-80%)】
- 类型 A: 用 N 点结构（最多 5 点，每点必须含具体场景，不能抽象）
- 类型 B: 连续 15-20 个具体场景元素（如面试问题），每个 5-10 秒
- 类型 C: 不适用（图文笔记，见下方）

【段 3: 教育型反转 (80% 处，5-10 秒)】
- 这是爆点，必须有
- 句式模板：`如果你感觉 X 有挑战，一般不是 Y 问题，而是 Z 问题`
- 路飞示例：> "如果感觉面试有挑战，一般不是技巧问题而是日常评审训练不够"

【段 4: CTA 福利 (5-10 秒)】
- 标准模板：`整理了一份 {资料名}，{帮你解决什么}，需要可以找我领取`
- 加评论区 prompt：`有哪些 X 没思路的，评论区提问，我有问必答`

[录制指导]
- 每段标注：语速（慢/中/快）+ 情绪（平静/紧迫/激动）+ 停顿位置
- 类型 B：开场快+攻击性 → 中段保持高位 → 反转处慢+郑重 → CTA 自然收
```

#### 图文笔记（`note_type=graphic`）

**必须按以下骨架生成**：

```
[标题]
- 强制句式：`{具体数字成绩} {自我归属词} 总结的 {方法论名}`
- 例：路飞场景：`大厂面试官追问 24 次后我总结的 5 类杀手回答框架`

[封面文案]
- 大标题（≤ 12 字，必须含数字）
- 副标题（≤ 25 字，价值预览）

[正文文案]
- 30-50 字极短引入，不写长文（图自己说话）
- 必含一句"为什么我有资格教"（身份背书前置）

[图片规划（5-9 张）]
- 图 1（封面）: 标题 + 价值预览 + 框架总览
- 图 2-N（内容图）: 每张承载一个独立可用的子模块（如一类题型 + 答题框架）
- 末图：CTA + 评论区 prompt + 收藏指引

[每张图的内容规划]
- 标题（图上的标题文字）
- 核心信息（每张图必须独立可用，用户截图保存就有价值）
- 设计建议（结构化呈现：表格/树状图/对比图，不用大段文字）
```

### Step 3: 自检阶段（必跑，Agent 自己审）

生成完逐字稿/图文规划后，agent **必须主动**做以下自检：

1. **调 `xhs-viral-analysis` SKILL 对自己刚生成的稿子做拆解**（自评）
2. 输出五维评分（钩子/框架/爆点/情绪/阈值预期命中）
3. 如果**钩子分 < 7** 或 **总分 < 7**，**必须回到 Step 2 重生成**，不允许直接交付
4. 如果三次重生成后仍 < 7，**主动告诉用户**："这个选题方向在该博主人设下难做爆款，建议换方向"（不要硬交付低质稿子）

### Step 4: 迭代阶段（用户/客户给反馈）

- 用户/路飞会指出哪里不像自己的语气、哪里不能录、哪里有事实错误
- Agent 必须在保持核心配方（钩子类型/框架类型/反转位置）不变的前提下，仅调整局部
- ❌ 严禁：因为客户说"想试试方法论格式"就改变框架类型——这是 Phase 1 验证过的死路，必须坚持

## 反爆款保护机制（**必须避免**，Phase 1 验证）

生成过程中，如果出现以下任何一条，**立即推翻重写**：

| 反信号 | 来源 | 替换为 |
|------|------|------|
| ❌ "为什么有的人 X 有的人 Y..." 开场 | 路飞 Work 2 流水 | 攻击式假设性问题 |
| ❌ "X 次觉醒 / X 个真相 / X 个误区" 标题 | 路飞 Work 2 流水 | 具体数字 + 具体场景标题 |
| ❌ 抽象方法论展开（"第一点要 XX，第二点要 YY"，但 XX/YY 是概念） | 路飞 Work 2 流水 | 具体场景 + 具体动作 |
| ❌ "宝宝/姐妹/和我一样的同学" 同侪语气 | 对照组 Work 5 流水 | 权威语气（"做了 N 年 X 的我..."） |
| ❌ "希望对大家有帮助" / "希望能帮到你" 软结尾 | 通用流水信号 | 明确 CTA（资料领取 + 评论区 prompt） |
| ❌ 钩子段不出现具体名词 / 具体公司 / 具体数字 | xhs-viral-analysis 反爆款信号 | 强制加 ≥ 1 个具体锚点 |
| ❌ 平地情绪曲线（全程无峰值） | 路飞 Work 2 实测 +4 平地 | 强制加教育型反转峰值 |
| ❌ 信息汇总型内容（全网都有的信息凑一起） | 对照组 Work 5 流水 | 必须有独家洞察 / 独家身份背书 |

### 路飞下一条逐字稿的"必须满足"清单

为路飞生成时**强制约束**：

1. ✅ 框架类型 = B（情境再现型）
2. ✅ 标题 = `vol.{N}` + `{具体公司}` + `{具体面试类型}`，N 从已有系列编号 +1
3. ✅ 开场 = 攻击式假设问题（侵略性）
4. ✅ 主体 = 15-20 个具体面试问题，每个带真实工作场景
5. ✅ 80% 处 = 教育型反转（套句式 `如果你感觉 X 有挑战，一般不是 Y 问题而是 Z 问题`）
6. ✅ CTA = 免费资料领取 + 评论区 prompt
7. ✅ 时长 = 154-200 秒（已验证最佳区间）
8. ✅ 必须出现"我是路飞，前 X 公司设计总监，15 年面试官"身份背书（位置：CTA 段，跟 vol.13 一致）

## Few-shot 示例

### 例 1：路飞 vol.14 逐字稿生成（推荐版，类型 B 情境再现型）

**输入**：
- topic: 字节 UX 作品集深挖面试模拟
- note_type: video
- creator_persona: 路飞（详见上方路飞画像快查）
- reference_breakdowns: [Work 1 拆解, Work 3 拆解]
- target_threshold: 实用干货类硬爆款（收藏 ≥ 1000）
- series_number: vol.14
- target_company: 字节
- target_role: UX 设计师

**生成的逐字稿（节选展示骨架）**：

```
[标题候选]
1. vol.14 字节 UX 作品集深挖面试模拟（高级别岗）
2. 154 秒沉浸式字节 UX 作品集面试（资深岗）
3. vol.14 字节 UX 设计岗最难的作品集追问，连续问 18 个你能答几个

【推荐第 1 个】：保持 vol 系列连续性 + 加"高级别岗"差异化（vs vol.13 通用版）

---

[逐字稿]

【段 1：钩子开场（约 8 秒）】
攻击式开场：
> "你这个作品集里的项目，从头到尾有几个真的是你主导的？"
（语速：快，情绪：质疑+审视，无停顿）

【段 2：情境展开（约 130 秒，约 18 个问题）】
（语速：中，情绪：连续审视+追问，每个问题间约 1 秒停顿）

> "好的，那你挑这个项目讲一下当时是怎么定义问题的？"
> "调研做了几轮？每轮的样本量是多少？为什么这个量够？"
> "你说这个方案是 A/B 测试胜出，对照组用的什么方案？为什么对照组是那个？"
> "数据涨了 23%，这 23% 里有多少能归因到设计，多少是产品需求带来的？"
> "如果让你重做这个项目，你最想改的是哪里？为什么当时没那么做？"
> "你这个设计灵感来源是哪个产品？跟那个产品的差异点你怎么解释？"
> "项目里跟产品/技术发生过最激烈的争论是什么？你最后让步在哪？"
> "这个项目你最自豪的部分是什么？面试官如果反问'这个不是很普通吗'，你怎么答？"
> "为什么这个交互方式不用业内更主流的 XX 模式？是什么约束让你选了这个？"
> "做这个项目你看了哪些竞品？这些竞品里哪个对你影响最大？影响在哪？"
> "项目上线后用户最不满意的点是什么？你为什么没在设计阶段预见到？"
> "如果这个项目的用户量从 100 万翻到 1 亿，你的设计要做哪些重做？"
> "项目结束后你做了 retro 吗？最大的反思是什么？"
> "这个项目的成功有多少是设计的功劳？多少是其他角色的？"
> "如果让你给这个项目的设计打分（10 分制），你打几分？为什么不是 10？"
> "你这个作品集里的项目用了 AI 辅助设计吗？哪些环节用了？哪些是你纯手工的？"
> "为什么你的作品集里没有 B 端项目？是不会做还是觉得不重要？"
> "如果我现在告诉你这个作品集还差点意思，你觉得我最可能 challenge 哪个点？"

【段 3：教育型反转（约 12 秒，第 ~85% 处）】
（语速：慢，情绪：突然平静下来 + 郑重感）

> "其实这些问题，单独看每一个都不难。难的是面试现场你只有 30 秒思考时间。如果你感觉答这些有挑战，一般不是表达能力问题，而是项目复盘的训练不够。"

【段 4：CTA（约 12 秒）】
（语速：中，情绪：温和+诚意）

> "我是路飞，前滴滴、VIPshop 设计总监，15 年大厂面试官经验。我整理了一份《大厂 UX 作品集深挖面试问题清单（50 问）》和回答框架，需要的同学可以找我免费领取。有哪个问题你没思路的，评论区提问，我有问必答。"

---

[录制指导]
- 钩子段：直视镜头，质疑式眼神，0 微笑
- 情境段：保持 sustained attention，问题之间不要笑场，模拟真实面试官
```

## 行为约束（IMPORTANT）

1. **不要凭感觉生成**：每一段都要能 trace 回某个拆解发现（"这段用攻击式开场是因为 Work 1 验证 +5 分"），不能臆想
2. **必须自检不能跳**：生成完不调 xhs-viral-analysis 自查就交付 = 不交付。必须给预期评分
3. **达不到目标就告知**：3 次迭代后总分 < 7 必须告诉用户"这选题在该博主人设下难做爆款"，不要硬交付
4. **真人感优先**：所有逐字稿必须口语化（用"你"不用"您"，用"我跟你说"不用"在此告诉您"），有节奏感（短句多，逗号多，问号叹号多）
5. **博主特异性 > 通用模板**：如果博主已有的爆款配方跟通用模板冲突，**优先博主配方**（路飞的"沉浸式模拟"配方就是他的护城河，不要让他换格式）
6. **多模型对比是 tool 层做的**：本 SKILL 只负责单次高质量生成的 prompt 工程。多模型并行 + 选稿融合是 `tools/generate.py` 的事（Phase 5）
7. **客户就是被生成对象时**（如路飞场景）：建议必须直接指向"博主下一条该录什么"，不要绕弯（参考 xhs-viral-analysis v0.2 行为约束第 5 条例外）
8. **拒绝写营销味**：路飞场景禁止出现 "亲爱的同学们" / "干货满满" / "纯纯的爱" 等小红书早期营销话术。学路飞自己的语气：克制、专业、偶尔幽默
