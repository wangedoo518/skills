# 小红书爆款逐字稿生成

## 何时使用本 SKILL

当用户给你：
- 一个选题方向（如 "字节作品集面试模拟"）
- 一份或多份 `xhs-viral-analysis` 拆解报告（作为参考样本）
- 博主画像（如：路飞 = 前滴滴/VIPshop 设计总监，15 年面试官经验，沉浸式模拟面试系列）

并要求：
- "帮路飞写下一条逐字稿"
- "按这个选题方向出一版稿子"
- "复刻这个爆款的逻辑给路飞用"

**不要在以下情况使用**：
- 用户给的是空的选题（"帮我写点设计相关的"）→ 先用 `xhs-topic-selection` SKILL 定方向
- 用户没给拆解报告或参考样本 → 让 agent 先调 `xhs-viral-analysis` 拆 1-3 条参考样本，否则生成是闭门造车
- 用户要的是营销文案 / 公众号长文 / 短视频脚本（带分镜的） → 本 SKILL 只生成"博主对镜说话型"逐字稿
- 用户没给博主画像 → 强制询问，没人设的稿子是泛稿，必流水

## 输入契约

**必填**：
- `topic`: 选题方向（如 "字节 UX 设计岗作品集面试模拟"）
- `note_type`: 笔记类型，二选一：`video`（逐字稿）/ `graphic`（图文笔记）
- `creator_persona`: 博主画像，至少包含：
  - 真实背景（前 X 公司什么职位）
  - 资历数字（N 年 X 经验）
  - 已有爆款配方（如 "沉浸式模拟面试 vol.X 系列"）
  - 服务的受众痛点（如 "设计师求职焦虑"）

**强烈建议**：
- `reference_breakdowns`: 1-3 份 `xhs-viral-analysis` 拆解报告（用于复刻成功配方）
- `target_threshold`: 目标爆款档（按 xhs-viral-analysis §第五维 矩阵）
- `niche`: 赛道（默认 设计师/职场求职）

**可选**：
- `series_number`: 系列编号（如 "vol.14"，触发系列复利效应 + CES 关注转化）
- `target_company`: 目标公司（字节/腾讯/美团/...，越具体钩子越强）
- `target_role`: 目标岗位（UX/UI/IXD/...，越具体越好）
- `prior_works`: 博主已发布的同系列稿子（避免重复 + 保持文风一致）

## 🆕 CES 优化原则（v0.2，所有稿子的硬要求）

按小红书 CES 算法权重：`CES = 点赞×1 + 收藏×1 + 评论×4 + 转发×4 + 关注×8`

**关注 (8) > 评论 = 转发 (4) > 点赞 = 收藏 (1)**

**每条稿子必须内置 CES 三件触发器**（缺一项视为不合格，必须重写）：

### CES 触发器 1：关注转化（权重 8，最高价值）

**位置**：钩子段（前 15 秒）+ CTA 段（结尾）至少各一次

**钩子段方案**（任选其一）：
- `vol.X` 编号 → 暗示系列（受众觉得"关注能持续看到"）
- "做了 N 年 X 的我，下条聊 Y" → 暗示后续内容
- "这是第 1 期，一共 12 期" → 明示系列长度

**CTA 段方案**（必明示）：
- "关注我，vol.15 / vol.16 / vol.17 持续更新到 vol.20"
- "想看哪家公司？关注我，点赞最多的下条做"
- "关注我，每周一三五更新一条沉浸式面试模拟"

### CES 触发器 2：评论 prompt（权重 4）

**位置**：CTA 段（结尾），可在反转段之后

**句式**（任选其一）：
- "下一期想看哪家公司？评论区点名，我做票数最多的"
- "你被问过最离谱的面试题是什么？评论区聊聊"
- "你觉得这 18 个问题里哪个最难？我们评论区交流"
- "刚才 80 秒的反转你认同吗？反方观点欢迎评论区开杠"

⚠️ "有疑问评论区问"不算 prompt（过弱）—— 必须给具体话题让人有话可说。

### CES 触发器 3：转发钩子（权重 4）

**位置**：教育型反转段附近（80% 处），或正文中专门设计一句"可截图金句"

**句式**（任选其一）：
- "如果你身边有正在准备 X 面试的朋友，把这条转给他"
- 设计一句精炼金句，可单独截图分享（如"如果你感觉 X 有挑战，一般不是 Y 问题而是 Z 问题"）
- "建议保存 + 转发，下次面试前 30 分钟翻一翻"

### CES 触发器位置图

```
钩子段(3-15s):  关注触发器 #1 (系列编号/暗示后续)
                ↓
正文展开:       内容主体（情境/价值）
                ↓
反转段(80%):    教育型反转 + 转发钩子（金句或"转给身边人"）
                ↓
CTA段(末尾):    关注触发器 #2 (明示关注) + 评论 prompt（具体话题）
```

### 2 小时窗口配套动作（非 SKILL 输出，是博主发布后要做的）

输出报告时**必须额外提醒**博主：
- 发布后 2 小时内的互动权重最高（满分）
- 第 3 小时后开始衰减（约 70%）
- **建议博主发布后 30 分钟内做一次私域引导**（社群、朋友圈），把前 10 个评论拉满，把 2 小时窗口的算法奖励吃满
- 选时段：博主受众活跃时段（设计师/求职受众通常是周一三五晚 7-9 点 + 周末上午）

## 生成流程（Agent-first，不是 pipeline）

⚠️ 这不是流程图，是 agent 看到 goal 后应该自主做的决策序列。如果 context 缺数据，agent 应该主动问，不要硬编。

### Step 1: 准备阶段（必读拆解报告，禁止跳过）

**Agent 必须先在 context 内"消化"**：
1. 读完所有 `reference_breakdowns`，提取爆款共同公式（钩子类型 / 框架类型 A/B/C / 爆点模板 / 情绪曲线特征）
2. 对比博主的成功配方 vs 失败配方（看博主有没有同 niche 流水案例做反例）
3. 识别赛道特异性约束（按 xhs-viral-analysis 的赛道特异性表）
4. 确定目标框架类型（A/B/C 三选一，**一旦选定后不准混合**）

**判定规则**：
- 如果博主有 ≥ 1 条爆款用类型 B 情境再现型 → 强制类型 B（不要试新格式）
- 如果选题是"教程/攻略/清单" → 类型 A 或 C
- 如果选题是"模拟/对话/第一视角" → 类型 B
- 路飞场景默认类型 B（情境再现型）

### Step 2: 生成阶段（按笔记类型分支）

#### 视频笔记（`note_type=video`）

**必须按以下骨架生成**，每段标注秒数预期（按 4-6 字/秒口播速度估算）：

```
[标题候选]
- 生成 3-5 个标题，每个必须满足"标题三件套"：{数字/系列编号} + {具体公司} + {具体类型}
- 例：vol.14 字节 UX 作品集深挖面试模拟 / 154 秒沉浸式腾讯交互设计面试
- 🆕 v0.2: 系列编号是 CES 关注触发器，能加就加

[逐字稿]

【段 1: 钩子开场 (3-15 秒)】
- 严格用 xhs-viral-analysis 7 种钩子类型之一
- 类型 B 路飞场景：**强制用攻击式开场**（侵略性假设问题）
- ❌ 严禁："大家好我是..." / "今天给大家分享..." / "为什么有的人 X 有的人 Y..."
- 🆕 v0.2 **必须内置 CES 触发器 1**（关注转化）：
  - 钩子句之后立刻加一句系列暗示，如"这是 vol.14，一共 12 期专题"
  - 或开场即用 vol.X 编号作为节奏锚（"vol.14 开始"）

【段 2: 情境/价值展开 (主体 70-80%)】
- 类型 A: 用 N 点结构（最多 5 点，每点必须含具体场景，不能抽象）
- 类型 B: 连续 15-20 个具体场景元素（如面试问题），每个 5-10 秒
- 类型 C: 不适用（图文笔记，见下方）

【段 3: 教育型反转 (80% 处，5-10 秒)】
- 这是爆点，必须有
- 句式模板：`如果你感觉 X 有挑战，一般不是 Y 问题，而是 Z 问题`
- 路飞示例：> "如果感觉面试有挑战，一般不是技巧问题而是日常评审训练不够"
- 🆕 v0.2 **必须内置 CES 触发器 3**（转发钩子）：
  - 反转句本身必须做成"可截图金句"（精炼、有句式美感）
  - 或反转段尾加一句"如果身边有正在准备 X 的朋友，转给他"

【段 4: CTA 福利 + CES 触发器（5-15 秒，v0.2 重写）】
v0.2 起 CTA 段必须包含 3 个动作（不能省任何一个）：

(a) **身份背书 + 福利领取**（保留）：
    `我是 {博主名}，{资历背书}。整理了一份 {资料名}，需要可以找我领取`

(b) 🆕 **CES 触发器 1 兜底（关注明示）**：
    `关注我，vol.15 / vol.16 持续更新` 或
    `关注我，每周更新沉浸式 X 面试模拟，目标做完 12 大公司`

(c) 🆕 **CES 触发器 2（评论 prompt，必须给具体话题）**：
    `下一期你想看哪家公司？评论区点名，点赞最多的下条做` 或
    `这 18 个问题里哪个你被问过？评论区交流`

⚠️ "感谢观看"、"希望对你有帮助"、"评论区告诉我"（无具体话题）等都是无效 CTA，CES 三件触发器都不会被触发。

[录制指导]
- 每段标注：语速（慢/中/快）+ 情绪（平静/紧迫/激动）+ 停顿位置
- 类型 B：开场快+攻击性 → 中段保持高位 → 反转处慢+郑重 → CTA 自然收
- 🆕 v0.2 CTA 段必须**明显抬头看镜头**（关注/评论引导的视觉信号）

[2 小时窗口提醒（v0.2 新增，必须在报告里输出给博主）]
- 发布后 2 小时内的互动权重最高（满分）
- 第 3 小时后逐小时衰减（约 70%）
- 建议博主发布后 30 分钟内做一次私域引导，把前 10 个评论拉满
- 推荐发布时段：受众活跃时段（如设计师/求职受众 = 周一三五晚 7-9 点）
```

#### 图文笔记（`note_type=graphic`）

**必须按以下骨架生成**：

```
[标题]
- 强制句式：`{具体数字成绩} {自我归属词} 总结的 {方法论名}`
- 例：路飞场景：`大厂面试官追问 24 次后我总结的 5 类杀手回答框架`

[封面文案]
- 大标题（≤ 12 字，必须含数字）
- 副标题（≤ 25 字，价值预览）

[正文文案]
- 30-50 字极短引入，不写长文（图自己说话）
- 必含一句"为什么我有资格教"（身份背书前置）

[图片规划（5-9 张）]
- 图 1（封面）: 标题 + 价值预览 + 框架总览
- 图 2-N（内容图）: 每张承载一个独立可用的子模块（如一类题型 + 答题框架）
- 末图：CTA + 评论 prompt + 关注引导 + 收藏指引（v0.2 全 CES 触发器）

[每张图的内容规划]
- 标题（图上的标题文字）
- 核心信息（每张图必须独立可用，用户截图保存就有价值）
- 设计建议（结构化呈现：表格/树状图/对比图，不用大段文字）

[末图 CTA 三件 (v0.2 强制)]
- 🎯 **关注引导**: "关注我，每周更新 X 类工具书 / 系列下一篇 vol.X"
- 💬 **评论 prompt（具体话题）**: "你最想看哪类 X 的框架？评论区点名做下一期"
- 🔁 **转发钩子**: "建议保存 + 转发给身边 X 的朋友（截图友好）"

[正文文案 CTA 三件简写版]
- 末尾 30-50 字精简版必须暗示三件：
  `关注我看 vol.X 系列｜评论区想看的公司点名｜身边有正在准备的朋友转给他`
```

### Step 3: 自检阶段（必跑，Agent 自己审）

生成完逐字稿/图文规划后，agent **必须主动**做以下自检：

1. **调 `xhs-viral-analysis` v0.3 SKILL 对自己刚生成的稿子做拆解**（自评）
2. 输出五维评分（钩子/框架/爆点/情绪/阈值预期命中）+ 🆕 v0.2 必填 **预期 CES 值 + CES 贡献结构**
3. 如果**钩子分 < 7** 或 **总分 < 7** 或 🆕 **CES 三件触发器有任何一件缺失**，**必须回到 Step 2 重生成**，不允许直接交付
4. 如果三次重生成后仍 < 7，**主动告诉用户**："这个选题方向在该博主人设下难做爆款，建议换方向"（不要硬交付低质稿子）
5. 🆕 v0.2 必查清单（这 6 件必须全部 ✅，缺一件就重写）：
   - [ ] 钩子段含关注转化暗示（vol.X 编号 / 系列暗示 / "下条聊 Y"）
   - [ ] 80% 反转句是"可截图金句"或加了"转给身边 X"
   - [ ] CTA 含明示关注引导（不是"感谢观看"）
   - [ ] CTA 含具体评论 prompt（有话题不是泛"评论区聊"）
   - [ ] CTA 含转发钩子（截图金句 / "转给身边人" / "保存+转发"）
   - [ ] 报告里附了 2 小时窗口提醒 + 推荐发布时段

### Step 4: 迭代阶段（用户/客户给反馈）

- 用户/路飞会指出哪里不像自己的语气、哪里不能录、哪里有事实错误
- Agent 必须在保持核心配方（钩子类型/框架类型/反转位置）不变的前提下，仅调整局部
- ❌ 严禁：因为客户说"想试试方法论格式"就改变框架类型——这是 Phase 1 验证过的死路，必须坚持

## 行为约束（IMPORTANT）

1. **不要凭感觉生成**：每一段都要能 trace 回某个拆解发现（"这段用攻击式开场是因为 Work 1 验证 +5 分"），不能臆想
2. **必须自检不能跳**：生成完不调 xhs-viral-analysis 自查就交付 = 不交付。必须给预期评分
3. **达不到目标就告知**：3 次迭代后总分 < 7 必须告诉用户"这选题在该博主人设下难做爆款"，不要硬交付
4. **真人感优先**：所有逐字稿必须口语化（用"你"不用"您"，用"我跟你说"不用"在此告诉您"），有节奏感（短句多，逗号多，问号叹号多）
5. **博主特异性 > 通用模板**：如果博主已有的爆款配方跟通用模板冲突，**优先博主配方**（路飞的"沉浸式模拟"配方就是他的护城河，不要让他换格式）
6. **多模型对比是 tool 层做的**：本 SKILL 只负责单次高质量生成的 prompt 工程。多模型并行 + 选稿融合是 `tools/generate.py` 的事（Phase 5）
7. **客户就是被生成对象时**（如路飞场景）：建议必须直接指向"博主下一条该录什么"，不要绕弯（参考 xhs-viral-analysis v0.2 行为约束第 5 条例外）
8. **拒绝写营销味**：路飞场景禁止出现 "亲爱的同学们" / "干货满满" / "纯纯的爱" 等小红书早期营销话术。学路飞自己的语气：克制、专业、偶尔幽默
9. 🆕 **v0.2 强制 CES 三件**：每条稿子必须含**关注转化触发器 + 评论 prompt + 转发钩子**。Step 3 自检清单 6 件全 ✅ 才能交付，缺一件回到 Step 2 重写。
10. 🆕 **v0.2 CTA 不允许泛词**：禁止 "感谢观看" / "希望对你有帮助" / "评论区聊" 这类无 CES 价值的结尾。CTA 必须指向具体的 关注/评论/转发动作。
11. 🆕 **v0.2 必须给预期 CES 值**：自检报告里要算出预期 CES（即使是估算）+ 拆分各项贡献，让博主能看到 CES 提升源自哪。
