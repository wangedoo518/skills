# 小红书选题方向推荐

## 何时使用本 SKILL

当用户/客户问：
- "我下周应该做什么选题？"
- "帮路飞推荐几个 vol.14 之后的选题方向"
- "这个赛道现在什么选题能爆？"
- "我擅长的内容跟市场热点怎么结合？"

**不要在以下情况使用**：
- 用户已经定好了选题方向，只是要写稿 → 直接用 `xhs-script-generation` SKILL
- 用户没给博主画像 → 必须先问，泛选题没意义（同一个选题对不同博主天差地别）
- 用户要的是"内容日历规划"（每天发什么）→ 本 SKILL 只做单次选题推荐，不做长期规划

## 输入契约

**必填**：
- `creator_persona`: 博主画像（同 xhs-script-generation 要求）。**最关键的是已验证的爆款配方**——博主已经爆过哪种格式
- `niche`: 赛道（设计师求职 / 美妆 / 穿搭 / ...）

**强烈建议**：
- `reference_breakdowns`: 1-N 份 `xhs-viral-analysis` 拆解报告（赛道近期爆款 + 博主自己的爆款/流水案例）。**没有这个 agent 是闭门造车**
- `prior_works`: 博主已发布作品清单（避免重复选题）
- `target_count`: 要几个候选选题（默认 3-5）

**可选**：
- `time_horizon`: 时效范围（"本周" / "本月" / "Q3 整季"）
- `hot_topics`: 当前热点（如 "字节裁员" "AI 设计工具更新" "校招开放"）
- `note_type_preference`: 偏好视频还是图文（默认两种都给）
- `constraints`: 博主拒绝的方向（如 "不做 AI 替代设计师话题，不政治正确"）

## 分析框架（Agent-first 决策模式）

⚠️ 这不是 pipeline，是 agent 在 context 里读到 goal 后该走的思考路径。

### Step 1: 博主能力定位（必跑）

从博主画像 + 拆解报告中提取：

1. **已验证爆款配方**：博主用什么钩子类型/框架类型/内容形态爆过？
2. **已验证流水配方**：博主用什么格式翻过车？（避雷）
3. **博主独家护城河**：背景/资历/差异化卖点（如路飞 = 15 年面试官 = 沉浸式模拟面试可信度）
4. **博主天花板**：当前爆款级别 vs 赛道天花板的差距

**输出**：博主能力雷达图（文字描述即可）

### Step 2: 赛道现状扫描

从 `reference_breakdowns` + 现有知识中提取：

1. **赛道近期爆款共性**：钩子类型分布 / 框架类型分布 / 主题分布
2. **赛道新兴趋势**：什么主题最近开始爆 / 什么主题在退潮
3. **赛道天花板**：最高互动的选题是什么级别（如 5w 收藏的"答题框架"）
4. **赛道空白区**：哪类需求有但供给不足（潜在蓝海）

### Step 3: 候选选题生成（≥ 2N，N = target_count）

按"博主配方 × 赛道需求"二维生成候选，每个候选必须有**起点假设**：

| 起点假设类型 | 例子（路飞） |
|------|------|
| **博主已爆配方 + 新场景** | 沉浸式面试模拟（已爆）+ 新公司/新岗位/新面试类型 |
| **博主已爆配方 + 新格式** | 沉浸式面试（已爆视频）→ 沉浸式职场对话（视频）/ 面试框架手册（图文） |
| **赛道爆款配方 + 博主独家素材** | 对照组 5.5w 收藏的"答题框架" + 路飞 15 年面试官独家整理 |
| **博主背景延伸 + 未做过的角度** | 路飞曾任设计总监 → "面试官视角的求职者最容易踩的坑" |
| **热点蹭法**（如有） | AI 工具更新 → "AI 取代设计师？面试官最想问的 3 个能力护城河" |

候选数量 **必须 ≥ 2 × target_count**，留 50% 余地做排序筛选。

### Step 4: 三维度评分（每个候选都打分）

| 维度 | 权重 | 评分依据 |
|------|------|---------|
| **博主匹配度** | 0.4 | 是否符合博主已验证配方 + 博主独家护城河发挥度 |
| **选题热度** | 0.4 | 赛道近期是否有同主题爆款 / 是否蹭得上热点 / 受众痛点强度 |
| **可执行性**（10 - 复刻难度） | 0.2 | 素材是否够 / 是否需要新场景 / 博主能否驾驭 |

**评分细则**：

#### 博主匹配度（0-10）
- 10: 跟博主已爆配方完全一致（如路飞做 vol.14 字节作品集面试）
- 8: 博主已爆配方 + 一个新变量（新公司/新岗位/新格式）
- 6: 博主未做过但能驾驭（背景匹配）
- 4: 博主能做但跟主线偏离
- 2: 博主能做但要重新建立人设
- 0: 跟博主背景完全冲突

#### 选题热度（0-10）
- 10: 赛道近期有同主题超大爆款（5w+ 收藏）+ 自带话题度
- 8: 赛道近期有硬爆款（1k+ 收藏）
- 6: 赛道常青选题（持续有中等爆款）
- 4: 小众主题但受众精准
- 2: 冷门主题
- 0: 完全没有市场信号

#### 可执行性（0-10，越高越好）
- 10: 博主直接能做，无新素材需求
- 8: 需要简单准备（如换个公司、换个面试类型）
- 6: 需要新素材但好获取（找几个新案例）
- 4: 需要新格式实验（如路飞做图文笔记第一次）
- 2: 需要外部资源协调（拍摄场景/嘉宾）
- 0: 当前条件做不了

**总分公式** = `0.4 × 博主匹配度 + 0.4 × 选题热度 + 0.2 × 可执行性`

### Step 5: 风险/可执行性校验（必跑）

每个候选都要识别：

1. **博主独家壁垒**：这个选题离开博主背景还成立吗？
2. **赛道竞争度**：是不是大家都在做？（红海预警）
3. **时效性**：选题有没有窗口期？（蹭热点的会失效）
4. **复刻路径**：博主下一步具体要做什么准备？
5. **测试成本**：失败的话损失什么？（首条新格式 = 高失败成本）

## 三个匹配维度详解

### 1. 博主匹配度的高分关键

判断"是否符合博主已验证爆款配方"时，按 xhs-viral-analysis 的**框架类型 + 钩子类型**双重对齐：

- 框架类型对齐：博主已爆 类型 B，新选题如果也是类型 B → 加分
- 钩子类型对齐：博主已爆 沉浸式体验型，新选题如果同类 → 加分
- 都对齐 = 几乎是稳妥升级版（同配方 + 新场景）
- 任一不对齐 = 需要博主跳出舒适区，至少要降 2 分

**护城河放大器**：博主有独家背景（如路飞 15 年面试官），选题如果**直接消费这个背景** → +2 分。

### 2. 选题热度的高分关键

不要靠"我觉得"，要 cite 拆解报告：

- "近期同主题 X 条爆款" → 强热度信号
- "赛道天花板是 X 主题 + Y 配方" → 复刻有路径
- "对照组同类选题最高互动 X" → 上限可见
- 没有任何参考样本支撑的热度评分 = 不可信，必须降分或标注 "热度信号不足"

### 3. 可执行性的扣分项

- 需要博主第一次尝试新格式 → -2 分（首条新格式试错成本高）
- 需要新素材采集 → -1 ~ -3 分（看采集难度）
- 需要打破博主已有人设 → -3 分（重新教育受众成本高）
- 蹭时效性热点 → -1 分（窗口期短，错过就废）

- 评分：博主匹配 7/10（新格式，未验证），热度 10/10（对照组 5.5w 验证有效），可执行 6/10（路飞首次做图文，需要试错 1-2 条）
- **总分：8.0/10**
- 风险：首次做图文 = 失败成本相对高；图片设计能力需要配合（不只是排版）
- 路飞要准备：5 张图的视觉设计 + 框架抽象能力

### 路飞场景的反爆款选题信号

⚠️ 见到下面这些主题/角度直接拒绝（来自 Phase 1 实证）：

- ❌ "X 次觉醒 / X 个真相 / X 个误区"型抽象方法论（Work 2 验证）
- ❌ "为什么有的人 X 有的人不"型说教式设问开场
- ❌ "宝宝/姐妹"型同侪语气（违反路飞专业人设）
- ❌ 信息汇总型（"2026 校招时间轴" "互联网大厂清单"）—— 没有路飞独家洞察的内容是流水
- ❌ 跟博主背景脱钩的选题（如让路飞做"美妆测评"——离设计/求职太远）
- ❌ 一次性热点（如某天的字节裁员新闻）—— 蹭热点窗口期短，跟系列复利不兼容

## Few-shot 示例

### 例 1：路飞 vol.14 之后的选题推荐（完整流程）

**输入**：
- creator_persona: 路飞画像（见 xhs-script-generation 路飞场景配方）
- niche: 设计师职业发展 + 大厂求职
- reference_breakdowns: [Work 1, 2, 3, 4, 5 拆解报告 ← Phase 1.1-1.2 产物]
- prior_works: [vol.1 - vol.13 沉浸式面试模拟系列]
- target_count: 3
- time_horizon: 本月（4 周）

**输出报告（节选）**：

```markdown
# 选题推荐：路飞设计沉思录（设计师求职垂类）

> 输入：博主画像 + 5 份拆解报告 + 时效范围=本月
> 目标：推荐 3 个选题方向
> 候选池：8 个候选 → 排序后筛出 Top 3

## 一、博主能力定位

- **已验证爆款配方**：沉浸式面试模拟（类型 B 情境再现型 + 攻击式开场 + 18-20 个具体面试问题 + 80% 处教育型反转 + 资料领取 CTA + vol.X 系列）。Works 1/3 双爆款验证（收藏 2697/3946）
- **已验证流水配方**：抽象方法论（"X 次觉醒"型，类型 A 但内容空洞）。Work 2 验证（收藏 30 = 流水）
- **独家护城河**：前滴滴、VIPshop 设计总监 + 15 年大厂面试官经验 + 辅导 500+ 同学进大厂的数据背书
- **当前天花板**：实用干货类硬爆款（收藏 3946）；距离赛道天花板（对照组 Work 4 收藏 5.5 万）还有 14 倍差距

## 二、赛道现状

- **近期爆款共性**：实用干货类收藏 > 点赞；钩子必含具体公司+具体岗位+数字
```

## 行为约束（IMPORTANT）

1. **不要凭"我觉得"出选题**：每个候选必须能 trace 回拆解报告里的具体爆款（或明说"无参考样本，凭赛道经验推断，可信度低"）

3. **博主匹配度 < 6 的候选 不能进 Top**：哪怕选题再热，博主驾驭不了就是泡沫
4. **可执行性 ≤ 4 的候选要明确警告**：博主可能要付出高成本，必须让他知情
5. **三维度评分必须独立**：不要让"热度"高分拉动"匹配度"分数（避免循环论证）
6. **总分公式不能改**：`0.4 × 匹配度 + 0.4 × 热度 + 0.2 × 可执行性`，固定公式才能横比
7. **保持选题库的同步**：本 SKILL 跟 `xhs-script-generation` 的"路飞场景配方"中的候选库**同步维护**。修改本 SKILL 候选时必须同步另一份
8. **不要替博主决定**：输出推荐排序后，**最终选哪个由用户/博主拍板**。SKILL 给依据，不下决定
9. **客户=博主本人时**（如路飞场景）：建议必须可操作（"路飞下一步要做的准备"必须是具体动作，不是"加强沉淀"这种废话）
